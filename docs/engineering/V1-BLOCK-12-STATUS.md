## Frozen product boundary

- Type selection authority is `user_only`; WriteStorm never infers or automatically assigns a type from source text.
- TypeLibrary version 1 contains seven MainType and seven ContentFocus identities. Books may be imported unassigned, may retain focus-only metadata, and require a MainType before formal analysis.
- Existing analysis snapshots are immutable. Classification CAS changes only current Book metadata; upgrades require a new snapshot and explicit impact plan.
- Prompt sample status, immutable publication fact, current published pointer, activation, and rollback operation remain separate concepts. Aggregate provenance is validated explicitly; publish only moves to a strictly larger `templateVersion`, while rollback only moves to a smaller historically published `templateVersion`.
- Technique Library remains a truthful non-persisted empty/read-only shell. SourceSnapshot never writes back to observations, candidates, EvidenceAnchors, review state, or source Books.
- Original shelf remains an independent non-creating placeholder and never displays Technique Library content.
- Settings remains non-executing: No real AI, connector, sample preview, publication mutation, health, repair, or privileged renderer path exists.
